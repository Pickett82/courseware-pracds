02. Anatomy of a Figure: Figures vs Axes¶

Figure = the whole figure

  • axes
    • 2D vs 3D
  • canvas
    • where everything is drawn
  • artists
    • anything drawn on the canvas
  • axes
    • 2D vs 3D
      • data is plotted within Axes
    • labels of the x and y axes
    • tick spacing
    • legends
    • etc
  • canvas
    • where everything is drawn
    • i.e. upon which plots shown, one or more plots
      • may contain several axes
    • effectively invisible by user
  • artists
    • anything drawn on the canvas
      • titles,
      • figure legends,
      • colour of the background,
      • etc
      • even the Figure, Axes, and Axis objects

03. API modes¶

  • two approaches to plotting with Matplotlib

    • Object Oriented API (follows the Python object model)

    • MATLAB-like API (aka pyplot API)

Further Reading:

  • https://matplotlib.org/api/index.html
  • https://matplotlib.org/tutorials/introductory/lifecycle.html

04. MATLAB-like API¶

  • NOT objects

    • this is within the implementation
      • EASIER TO USE
      • Flexibility - lacking

  • functions

    • affect a figure
      • creates a figure
      • creates a plotting area in a figure
      • plots some lines in a plotting area
      • decorates the plot with labels
      • etc

Changing Appearances: Adding Legends¶

In [8]:
plt.plot(X, Y1,   label = 'sin(x)'  )
plt.plot(X, Y2,   label = 'cos(x)'  )
plt.title('sin(x) and cos(x)')
plt.xlabel('values of x')
plt.ylabel('f(x)')

plt.legend(); # add a legend, (auto-generate using the axis labels)

[ASIDE] can specify legend:

  • either, at artist creation (e.g. plt.plot(X, Y, label='sin(x)'))
  • or, by calling the set_label() method
