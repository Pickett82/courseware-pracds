Nearest Neighbours¶

$f(x) = y_i, i = \text{argmin}_j || x_j - x||$¶

  • (closest = notion of distance)
$$d(x_{i}, x_{l}) = \sqrt{(x_{i1} - x_{l1})^2 + (x_{i2} - x_{l2})^2 + ... + (x_{ip} - x_{lp})^2}$$
  • assumes that points that are close to one another are similar
  • neglects a lot of information
  • wont help you understand the drivers

from sklearn.metrics import euclidean_distances
from sklearn.neighbors import KNeighborsClassifier

def plot_n_neighbors(X, y, n_neighbors=1):
    X_test = np.array([[0.5, 3], [4, 2], [-.5, .5]])
    dist = euclidean_distances(X, X_test)
    closest = np.argsort(dist, axis=0)

    plt.figure()
    plt.gca().set_aspect("equal")
    for x, neighbors in zip(X_test, closest.T):
        for neighbor in neighbors[:n_neighbors]:
            plt.arrow(x[0], x[1], X[neighbor, 0] - x[0],
                    X[neighbor, 1] - x[1], head_width=0, fc='k', ec='k')

    clf = KNeighborsClassifier(n_neighbors=n_neighbors).fit(X, y)
    test_points = plt.scatter(X_test[:, 0], X_test[:, 1], c=clf.predict(X_test), marker="*", s = 100)
    training_points = plt.scatter(X[:, 0], X[:, 1], c=y)

def plot_2d_classification(classifier, X, fill=False, ax=None, eps=None, alpha=1):                                       
    # multiclass                                                                                                                 
    if eps is None:                                                                                                              
      eps = X.std() / 2.                                                                                                       

    if ax is None:                                                                                                               
      ax = plt.gca()                                                                                                           

    x_min, x_max = X[:, 0].min() - eps, X[:, 0].max() + eps                                                                      
    y_min, y_max = X[:, 1].min() - eps, X[:, 1].max() + eps                                                                      
    xx = np.linspace(x_min, x_max, 1000)                                                                                         
    yy = np.linspace(y_min, y_max, 1000)                                                                                         

    X1, X2 = np.meshgrid(xx, yy)                                                                                                 
    X_grid = np.c_[X1.ravel(), X2.ravel()]                                                                                       
    decision_values = classifier.predict(X_grid)                                                                                 
    ax.imshow(decision_values.reshape(X1.shape), extent=(x_min, x_max,                                                           
                                                       y_min, y_max),                                                          
            aspect='auto', origin='lower', alpha=alpha)                                                               
    ax.set_xlim(x_min, x_max)                                                                                                    
    ax.set_ylim(y_min, y_max)                                                                                                    
    ax.set_xticks(())                                                                                                            
    ax.set_yticks(())

# k = 1 => overfit / complex; k = 30 => decision boundary is simple, but many pts incorrectly labelled

k = small

  • noise will have a higher influence on the result

k = large

  • computationally expensive
  • defeats the basic philosophy behind KNN (that points that are near might have similar densities or classes)

kNN

  • non parametric
    • no assumption for underlying data distribution
    • model structure determined from the dataset (i.e. usually what happens IRL)
      • data does not obey a theoretical structure (e.g. gaussian etc)
  • lazy
    • does not "need" training data points for model generation
      • minimal/zilch training phase --> costly testing (wrt time and memory)
      • i.e. wait "till last second" before classifying
  • accuracy degraded if,
    • noisy or irrelevant features (solution: need lots of data, or feature selection/PCA)
    • feature scales are not consistent with their importance (solution: use of evolutionary algos)
k is NOT learnt from data, decided by USER during model learning/fitting¶
